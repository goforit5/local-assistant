## ğŸ“ Metadata Structure

Each document includes YAML frontmatter with standardized metadata:

```yaml
---
doc_id: cc-section-topic
section: Section Name
title: Document Title
description: Brief description (50-80 characters)
status: complete | missing
last_updated: YYYY-MM-DD
---
```

**Metadata Fields**:
- `doc_id`: Unique identifier following `cc-{section}-{topic}` pattern
- `section`: Top-level section name (Getting Started, Build with Claude Code, etc.)
- `title`: Document title as it appears in navigation
- `description`: Brief description for search and navigation
- `status`: `complete` (documentation available) or `missing` (placeholder)
- `last_updated`: Date of last update in YYYY-MM-DD format
